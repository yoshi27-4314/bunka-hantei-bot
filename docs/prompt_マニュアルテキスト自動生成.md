## 今回やりたいこと

### 1. HTMLからテキストを自動抽出するスクリプト（自動変換プログラム）を作る

**ルール：HTMLが正（マスター）、テキストは自動生成。手で触らない。**

```
docs/分荷判定_作業マニュアル.html  ← 正（手動編集する）
        ↓ スクリプト実行
docs/help/分荷判定.txt             ← 自動生成（手で触らない）
```

### 対象HTMLファイル（4つ + 今後追加される可能性あり）
| HTML | 出力先テキスト |
|---|---|
| docs/分荷判定_作業マニュアル.html | docs/help/分荷判定.txt |
| docs/商品撮影_作業マニュアル.html | docs/help/商品撮影.txt |
| docs/出品保管_作業マニュアル.html | docs/help/出品保管.txt |
| docs/梱包出荷_作業マニュアル.html | docs/help/梱包出荷.txt |

### テキストの構造要件

ヘルプ機能でAIに渡すため、以下の構造にする：

```
# セクション名（h1に対応）

## 小見出し名（h2に対応）

本文テキスト（HTMLタグ除去、装飾除去、純粋なテキストのみ）

---

# 次のセクション名
...
```

- HTMLタグ（装飾用の記号）を全て除去
- CSSスタイル（見た目の設定）を全て除去
- 見出し階層（h1 → `#`、h2 → `##`、h3 → `###`）を Markdown（マークダウン = テキストの書式記法）に変換
- テーブル（表）はそのままテキストで再現（`|` 区切り）
- コードブロック（プログラムの表示枠）の中身はそのまま保持
- 画像・装飾用の絵文字は除去しない（スタッフ向けの表示に使うため残す）

### 2. 変換スクリプトの要件

- Python（パイソン = プログラミング言語）で書く（app.pyと同じ環境で動かすため）
- ファイル名: `generate_help_texts.py`（プロジェクトルートに配置）
- 実行方法: `python generate_help_texts.py` で全HTMLを一括変換
- `docs/help/` フォルダがなければ自動作成
- HTMLファイルが追加されたら、スクリプト内のマッピング（対応表）に追記するだけで対応できる設計
- 外部ライブラリ（追加パッケージ）: `beautifulsoup4`（HTMLを解析するライブラリ）を使用可

### 3. ルール（絶対厳守）
- コードを1行でも変更したら CLAUDE.md の変更ログに記録すること
- 生成されたテキストが正しいか、各ファイルの見出し数・本文量を確認すること
- `docs/help/*.txt` は `.gitignore`（Git管理から除外するリスト）に入れない（リポジトリに含める。Railwayで使うため）

---

まず `docs/` 内のHTMLファイルを読んで構造を把握し、スクリプトを作成してください。作成後、全HTMLを変換して結果を確認してください。
