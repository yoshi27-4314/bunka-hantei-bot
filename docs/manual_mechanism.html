03
🧵
スレッド方式によるセッション管理
複数の案件を同時並行処理しても混乱しない仕組み
🔑 仕組みの核心
Slackの「新規投稿」がセッションの親になり、
その返信（スレッド）はすべて同じ案件に紐づく。
コマンドはスレッド内でのみ有効なので、
別の案件の操作が混入することがない。
イメージ
📦 [新規] 商品A の写真
  └ 💬 第一　→ 管理番号2603-0001 発行
  └ 💬 A 正常動作　→ 動作確認完了
📦 [新規] 商品B の写真
  └ 💬 第二　→ 管理番号2603-0002 発行
  └ 💬 B 傷あり　→ 動作確認完了
♻️
同時並行処理
商品Aの処理中に
商品Bを別スレッドで
始めることができる。

混乱なし・混入なし
