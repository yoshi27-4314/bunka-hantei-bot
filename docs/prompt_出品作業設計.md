### 前工程（実装済み・稼働中）
1. **分荷判定** → AIが商品を分析、販売チャンネル（eBayシングル/eBayまとめ/ヤフオクビンテージ/ヤフオク現行/ヤフオクまとめ/ロット販売/社内利用/スクラップ/廃棄）を提案 → スタッフが確定

4. **商品撮影** → 撮影チャンネル（白洲次郎）でテプラ写真→商品写真→完了。Google Driveに自動保存

### 出品作業の現状（岩崎弥太郎チャンネル・実装済みだが改善が必要）

**現在のフロー（app.py の出品保管チャンネル処理）：**
1. テプラ写真 or 管理番号テキストを新規投稿 → セッション開始（作業の一連の流れが始まる）
2. Monday.comからデータ取得 → Claude（Haiku）で出品タイトル・説明文・開始価格を自動生成
3. 修正コマンドで調整（`タイトル：` / `開始価格：` / `説明文：` / `サイズ：`）
4. `ページ作成完了` → Monday.comのステータスが更新される
5. ロケーション番号（A-12など）入力 → 出品登録完了 → Monday.com「出品待ち」に更新

**関連ファイル：**
- `app.py` — Slackのイベント処理（メッセージ受信）・出品セッション管理（作業中データのメモリ上での一時保持）・タイトル/説明文の自動生成・完了処理
  - 主な関数: `handle_shuppinon_channel()` / `generate_listing_content()` / `post_listing_summary()` / `execute_listing()`
- `yahooauction_sheet.gs` — GAS（Google Apps Script = Googleスプレッドシート上で動くプログラム）。出品管理シートの操作、AIタイトル/説明文生成、CSV出力、送料表
- `webhook.gs` — GAS。app.pyから送られてきたデータをシートに書き込むエンドポイント（外部からデータを受け取る入口）
- `docs/出品保管_作業マニュアル.html` — 現行の操作マニュアル（V1）
- `CLAUDE.md` — プロジェクト全仕様書（変更時は必ず更新すること）

**GAS（yahooauction_sheet.gs）の主要機能：**
- 出品管理シート: CSV出力用のメインシート（管理番号/タイトル/説明文/開始価格/画像URL/送料…）
- スプレッドシートのメニュー: DBから商品追加 / タイトル生成（AI）/ 説明文生成（AI）/ ヤフオクCSV出力
- 送料表: 佐川（契約価格＋定価）/ 西濃ミニ便 / アートデリバリー
- 出品担当マーク: KH=〇（林）/ YY=▽（横山）/ TK=◇（鶴岡）→ タイトル先頭に付与
- オークション設定: 出品期間4日 / 終了22:00 / 自動再出品3回 / 返品不可 / 発送元:岐阜県

### ルール（絶対厳守）
- コードを1行でも変更したら CLAUDE.md の変更ログに記録すること
- 推測で仕様を書かない。必ず app.py の実コードを根拠にすること
- 全API呼び出しにtry-except（エラーが起きても止まらないようにする処理）を入れること
- APIキー・トークンはコードに直書きしない
- Anthropic API は Tier1。レートリミット（呼び出し回数制限）は5RPM（1分に5リクエストまで）。429エラー（制限超過の応答コード）時はRetry-Afterヘッダー（何秒待てばいいかの指示）を守ること

### 7. 新しい声ポイント制度

- 制度名: 新しい声ポイント制度
- コマンド: `新しい声 ●●`（`あたらしいこえ` も対応）。DM（ダイレクトメッセージ）専用
- カテゴリ: 要望 / アイデア / 相談（AIが自動判別）
- 記録先: Monday.com新ボード（気づきボード）
- ステータス検知: ウェブフック（Webhook = 外部からの通知を受け取る仕組み。メイン）＋ ポーリング（Polling = 定期的に問い合わせる方式。1日1回 朝8:50、補助）

**通知先:**
- DM（本人）: 全ステータス変更・ポイント通知
- #新しい声（浅野専用チャンネル）: 全件の内容表示 + スレッド（Thread = 返信の連なり）で経過更新
- #社内連絡（全員）: 毎朝8:55まとめ通知。動きがない日は通知なし
  - 投稿あり（匿名）、受理（匿名）、実装（内容・実名・獲得ポイント）、優秀（内容・実名・獲得ポイント）

**受理の3パターン:**
- 受理(+1pt) → 採用(+3pt) → 実装(+5pt) に進む
- 見送り（追加なし、投稿1ptは残る）
- 却下（投稿1ptも取消 = 0pt）

**ポイント:** 投稿+1 / 受理+1 / 採用+3 / 実装+5 / 優秀+20
**換金レート:** 1pt = 100円
**支払い:** 四半期（集計: 12-2月/3-5月/6-8月/9-11月 → 支払い: 3/6/9/12月給与に加算）
**ポイント確認:** DMで `ポイント` と入力
**名前:** 記名制
