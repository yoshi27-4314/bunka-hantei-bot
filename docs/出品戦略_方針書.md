## 3. ロードマップ（5テーマ）

### 依存関係

```
[テーマ1] AI生成レベル向上 ← 全ての土台
    ↓
[テーマ2] ヤフオクビジネスアカウント + 自動出品
```

### テーマ1：AI生成レベル向上（最優先）

**目標：Botが生成した出品データを修正なしでそのまま出品できる状態にする**

#### 現状の課題

| # | 課題 | 影響 |
|---|------|------|
| A1 | `generate_listing_content()` に渡す情報が少ない（アイテム名・メーカー・品番・状態・写真なし） | 生成品質が低く修正が多い |
| A2 | app.py(Haiku)とGAS(Opus)で説明文生成が二重実装 | 品質にばらつき。送料・免責文がapp.py側にない |
| A3 | タイトル40文字制限（正しくは65文字） | 検索キーワードが不足 |
| A4 | 出品担当マーク未付与（GAS側のみ実装） | タイトル先頭のマークが欠ける |
| A5 | 送料テーブル・免責文が説明文に含まれない | 出品後に手動追記が必要 |
| A6 | conditionフィールドの混乱（確信度を入れている） | 商品状態が正しく記録されない |
| A7 | ロケーション番号のバリデーションなし | 雑談テキストで誤確定するリスク |
| A8 | 免責文（DISCLAIMER）に文字化け・誤字あり | そのまま出品すると品質問題 |
| A9 | webhook.gsで出品ステータスが即「出品中」になる | 実際はまだ出品していない |
| A10 | Drive画像URLが出品管理シートの画像1〜5に連携されない | CSV出力時に画像なし |

#### 実装計画

**Phase 1A — データ品質修正（即時）**

| # | 対応 | ファイル |
|---|------|---------|
| 1 | `generate_listing_content()` にアイテム名・メーカー・品番・状態・Drive画像URLを追加 | app.py |
| 2 | タイトル文字数を65文字に修正 + 担当マーク対応 | app.py |
| 3 | conditionを確信度→商品状態に修正 | app.py |
| 4 | ロケーション番号バリデーション追加（`A-12` 等のパターンマッチ） | app.py |
| 5 | 出品ステータス「出品中」→「出品待ち」に修正 | webhook.gs |
| 6 | 免責文（DISCLAIMER）の文字化け修正 | yahooauction_sheet.gs |

**Phase 1B — 説明文品質向上**

| # | 対応 | ファイル |
|---|------|---------|
| 7 | app.pyの説明文生成にテンプレート・送料テーブル・免責文を統合 | app.py |
| 8 | アカウント区分（V/G/M）に応じたテンプレート自動選択 | app.py |
| 9 | Drive画像URLを出品管理シートの画像1〜5に自動転記 | webhook.gs |
| 10 | app.pyとGASの説明文生成を統一（app.py側に寄せる） | app.py / yahooauction_sheet.gs |

### テーマ2：ヤフオクビジネスアカウント + 自動出品

**前提：ビジネスアカウント取得後**

#### アカウント振り分けロジック

```
分荷判定の確定チャンネル → 出品アカウント自動振り分け

  ヤフオクビンテージ   → アイロンポット
  ヤフオク現行           → ブロカント
  ヤフオクまとめ         → まとめ売り
  委託（渡辺質店）       → 委託1アカウント
  委託（ビッグスポーツ） → 委託2アカウント
  eBayシングル           → eBay（テーマ4で対応）
  eBayまとめ             → eBay（テーマ4で対応）
```

#### 出品フロー（ビジネスアカウント後）

```
Bot生成（高品質データ）
  → アカウント別にCSV自動生成
  → Slackにダウンロードリンク通知
  → ヤフオク公式ストアCSVでインポート（簡易出品）
  → 出品完了をBotに通知 → ステータス自動更新
```

#### 必要な実装

| # | 対応 |
|---|------|
| 1 | app.pyにアカウント振り分け定数（ACCOUNT_ROUTING）追加 |
| 2 | GAS exportYahooCSV() をアカウント別に出力できるよう改修 |
| 3 | ストア公式CSVフォーマットへの対応（現在はオークタウンVer.1.17準拠） |
| 4 | Slack通知で「○件のCSVが出力されました」→ ダウンロードリンク |

### テーマ3：委託品のBot連携

**前提：テーマ1・2が整ってから着手**

#### 設計方針

- 分荷判定チャンネルに「委託元」属性を追加
- 投稿時に `渡辺` `ビッグ` 等のキーワードで識別 or 委託専用チャンネル
- 確定後、委託元ごとのMonday.comボードに自動登録
- 渡辺質店の先方管理番号はタックシールOCR or 手入力で紐付け

#### 渡辺質店の価格チェック自動化

```
現在（手動）:
  先方の希望売価を受け取る → 人間がヤフオク相場を調べる → 乖離が大きければ連絡

Bot連携後:
  先方の希望売価を入力 → AIがヤフオク落札相場と比較
  → 妥当：そのまま採用
  → 乖離大：「先方希望 ¥30,000 → 相場 ¥8,000〜12,000。要連絡」フラグ
  → 連絡テンプレート自動生成（在庫スペース負担を理由とした価格変更依頼文）
```

#### 月次精算レポート自動化

| 委託元 | 計算ロジック |
|--------|------------|
| 渡辺質店 | (落札価格 - ヤフオク手数料) × 先方指定比率 = 支払額 |
| ビッグスポーツ | (落札価格 - ヤフオク手数料) × 50% = 支払額 |

- スプレッドシートに精算シートを自動生成
- 月末に「精算レポート完成」をSlack通知
- 渡辺質店：現金支払い額の確認用
- ビッグスポーツ：振込額の確認用

### テーマ4：eBay自動出品

**前提：テーマ1が完了し、自社ビンテージ商品から開始**

#### 方針

- SAATS Commerce等の外部ツールに依存せず、Bot内で完結する設計を先に作る
- ヤフオク出品データ（日本語）→ Claude翻訳 → eBay出品データ生成

#### 必要な実装

| # | 対応 |
|---|------|
| 1 | 英語タイトル・英語説明文の自動生成（Claude） |
| 2 | JPY → USD変換（為替レート取得） |
| 3 | eBay固有フィールド（Item Specifics, Condition, Shipping Policy） |
| 4 | eBay出品CSV生成（File Exchange形式） |
| 5 | 国際送料テーブル（日本郵便 / FedEx / DHL） |

#### 対象商品の優先順位

1. ビンテージ・アンティーク（海外需要が高い）→ アイロンポット在庫
2. ブランド品（渡辺質店委託）
3. 希少な書籍（テーマ5と連動）

### テーマ5：書籍販売の検証（独立プロジェクト）

**何万冊は通常の分荷判定フローに乗らない。専用ワークフローが必要。**

#### Phase 5A — トリアージ（仕分け）の仕組み構築

最重要。何万冊を全部個別出品するのは不可能なので、効率的に仕分ける。

```
ISBNバーコードスキャン
  → Amazon/ヤフオク/eBayの価格を自動取得
  → 自動仕分け:
      A: 高値（1冊1,000円以上）→ 個別出品（Amazon FBA or ヤフオク）
      B: 中値（300〜1,000円）  → ジャンル別セット販売（ヤフオクまとめ）
      C: 低値（300円未満）     → まとめ売り or 古紙回収
      D: 希少本・洋書          → eBay
```

#### Phase 5B — 販路別出品

| 販路 | 対象 | 方式 |
|------|------|------|
| Amazon FBA | Aランク書籍 | ISBNで即出品。FBA倉庫に送れば発送・カスタマー対応不要 |
| ヤフオク | Aランク（Amazon非掲載）+ Bランク（セット） | Bot経由でCSV出品 |
| eBay | Dランク（希少本・洋書） | テーマ4のeBayフローを流用 |

#### Phase 5C — 検証ステップ（小規模スタート）

| ステップ | 内容 | 規模 |
|---------|------|------|
| 1 | 100冊をランダムに抜き取りISBNスキャン | 1日 |
| 2 | A/B/C/D分布を確認。高値本の比率を把握 | — |
| 3 | Aランク10冊をAmazon FBAで試験出品 | 2週間 |
| 4 | Bランク20冊をヤフオクセット販売で検証 | 2週間 |
| 5 | 結果を見て本格展開の判断 | — |

#### 必要なツール・環境

- ISBNバーコードリーダー（スマホアプリで可）
- Amazon出品アカウント（FBA利用）
- 書籍相場API（Amazon Product Advertising API等）
