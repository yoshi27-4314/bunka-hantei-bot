# Monday.com タスク管理システム 設計書

## 全体構成

```
各ワークスペースのタスクボード（部門の共有タスク）
　↓ Make.comで自動同期
個人プライベートボード（よしさん＋本人だけ）
　↓ Make.comで自動同期
よしさんのボード（全員分を一覧 ＋ プライベートタスク）
```

## 2. 共通カラム構成

| カラム名 | タイプ | 全ボード共通 | 説明 |
|---|---|---|---|
| タスク名 | text | ○ | やること |
| 担当者 | people | ○ | 誰がやるか |
| ステータス | status | ○ | 未着手→対応中→完了 |
| 期限 | date | ○ | いつまでか（過ぎると自動で赤） |
| 優先度 | status | ○ | 高・中・低 |
| プロジェクト名 | text | ○ | チームプロジェクト（空欄OK） |
| 顧客・案件 | text | ○ | 紐づく顧客や案件名 |
| メモ | long_text | ○ | 補足情報 |
| 作成日 | creation_log | ○ | 自動記録 |
| 最終更新 | last_updated | ○ | 自動記録 |

### 事業部ごとの追加カラム

| 事業部 | 追加カラム | タイプ |
|---|---|---|
| CM | 案件の種類 | dropdown |
| CM | 金額 | numbers |
| TB | 管理番号 | text |
| TB | 工程 | status |
| AIX | 種類（開発/打合せ/提案） | dropdown |
| 経営管理 | 分類（人材/資金/新規/仕入先） | dropdown |

### 個人ボードの追加グループ：KPI

| カラム名 | タイプ | 説明 |
|---|---|---|
| KPI名 | text | 撮影完了数、平均リードタイムなど |
| 目標 | numbers | 今月の目標値 |
| 実績 | numbers | 実績値（Bot連携で自動更新可能） |
| 達成率(%) | formula | 実績÷目標×100（自動計算） |

## 3. ステータスラベル

| ステータス | 色 | 意味 |
|---|---|---|
| 未着手 | グレー | まだ手をつけていない |
| 対応中 | 青 | 今やっている |
| 社外：相手待ち | オレンジ | お客様や取引先の返事待ち |
| 社内：対応必要 | 赤 | 自分たちの都合で止まっている |
| 完了 | 緑 | 終わった |

### 優先度ラベル

| 優先度 | 色 |
|---|---|
| 高 | 赤 |
| 中 | オレンジ |
| 低 | 青 |

## 4. 同期の仕組み（Make.com）

### 個人ボード → 事業部ボード（双方向同期）

```
スタッフが個人ボードでステータスを「完了」に変更
　↓ Make.com（トリガー: ステータス変更）
事業部のタスクボードでも「完了」に自動反映
```

```
事業部ボードに新しいタスクが追加される
　↓ Make.com（トリガー: 新規アイテム作成）
担当者の個人ボードに自動コピー
```

### 事業部ボード → よしさんのボード（一方向同期）

```
各事業部のタスクボードの更新
　↓ Make.com
よしさんの全体タスク管理ボードに自動反映
```

## 5. ダッシュボード

### よしさん用ダッシュボード
- 全事業部のタスク数（ステータス別）
- 期限切れ件数（大きく表示）
- 「社内：対応必要」の件数（赤で強調）
- スタッフ別の進捗
- 今月のKPIサマリー

### スタッフ用ダッシュボード（個人ボード内）
- 自分のタスク一覧（カンバン表示）
- 今月のKPI達成率
- 期限が近いタスク

---

## 6. 実装手順

1. 各ワークスペースにタスクボードを作成（API）
2. 個人プライベートボードを作成（API）
3. よしさんの全体タスク管理ボードを作成（API）
4. ステータス・優先度ラベルを設定（API）
5. カラムを作成（API）
6. KPIグループを個人ボードに追加（API）
7. Make.comで同期シナリオを作成（手動）
8. ダッシュボードを作成（API or 手動）
9. Slack通知連携（Make.com）
