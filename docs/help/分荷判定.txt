# 1　この仕組みの目的


## TakeBack事業とは


TakeBack事業は、法人や個人から不要になった中古品を買い取り、適切な販路で販売することで利益を生む事業です。


  「どこで売るか」によって、同じ商品でも売れる金額が大きく変わります。

  この「どこで売るか」を決める作業が 「分荷判定」 です。


## 分荷判定とは何をする作業か


- 受け取った商品の写真またはテキストを分荷判定チャンネルに投稿する
- AIが商品を分析し、最適な販売チャンネルの候補を提示する
- スタッフが内容を確認し、チャンネルを確定する
- 確定と同時に管理番号の発行・データ記録が自動で行われる


## あなたの作業が事業全体に与える影響


✅ 正確に判定・確定すると
    
- 適切な価格で早く売れる
- 会社の利益が確保される
- データが蓄積されAIの精度も上がる
- 次の仕入れ資金が回る

  
  
    ⚠️ 判定が不正確だと
    
- 本来高く売れた商品が安く出品される
- 在庫が長期化して保管コストが発生
- 記録が不正確になり原価計算に影響


💡 一つひとつの判定が、チーム全体の成果につながっています。「わからない」と感じたら、迷わず確認してください。


## 作業の全体像


```
商品が届く
　　↓
📱 分荷判定チャンネルに写真（またはテキスト）を投稿
　　↓
🤖 AIが自動で分析・商品確認
　　↓
👤 スタッフが確認（合っていれば「はい」）
　　↓
🤖 AIが状態確認を依頼（通販チャンネル候補の場合のみ）
　　↓
👤 スタッフが状態ランク（S/A/B/C/D）を入力
　　↓
🤖 AIが判定結果・予想価格・スコアを出力
　　↓
👤 スタッフが「第一」「第二」またはチャンネル名で確定
　　↓
✅ 管理番号が発行される（通販チャンネルのみ）
✅ データが自動で記録される
✅ Monday.comに在庫登録される（通販チャンネルのみ）
```

# 2　販売チャンネルとは何か


## チャンネルの種類（全9種類）


「チャンネル」とは、商品をどこで売るかという「販路」のことです。

  商品の種類・状態・価格帯によって、最も利益が出やすいチャンネルが異なります。


📋 通販チャンネル（管理番号が発行される）
    
| チャンネル名 | どんな商品向け |
| --- | --- |
| eBayシングル | 海外需要のある希少品・ヴィンテージ |
| eBayまとめ | 単品では安いが複数まとめると価値が出る品 |
| ヤフオクヴィンテージ | 昭和レトロ・アンティーク・コレクター品 |
| ヤフオク現行 | 現行モデルの家電・家具・日用品 |
| ヤフオクまとめ | 単品では安い商品を複数まとめて出品 |

  
  
    📦 非通販チャンネル（管理番号は発行されない）
    
| チャンネル名 | どんな商品向け |
| --- | --- |
| ロット販売 | 業者へまとめて一括売却する商品 |
| 社内利用 | 事業内で使用する備品・消耗品 |
| スクラップ | 素材・金属としての価値のみある品 |
| 廃棄 | 販売・利用が難しく処分が必要な品 |

    ※ データベースには記録されますが、管理番号は発行されません。また状態確認のステップもスキップされます。


## チャンネルを選ぶときの考え方


AIが第一候補・第二候補を提示しますが、最終的な判断はスタッフが行います。

  迷ったときは以下の順番で考えてみましょう。
  
- 海外やコレクターに需要がありそうか？ → eBay・ヤフオクヴィンテージを検討
- 国内で普通に売れそうか・需要が安定しているか？ → ヤフオク現行を検討
- 単品では安いが、まとめると価値が出そうか？ → まとめ系を検討
- ①〜③に当てはまらない場合 → ロット販売・スクラップ・廃棄を検討

  
    💡 判断に迷う場合は `再判定` コマンドで追加情報を入力するか、先輩・責任者に相談してください。

# 3　分荷作業の流れ（8ステップ）


作業場所：分荷判定チャンネル（北大路魯山人）

  商品の写真またはテキストをこのチャンネルに投稿すると、AIが自動で判定を始めます。


## ステップ1　商品情報を投稿する


分荷判定チャンネルに、以下のどちらかで投稿します。
  
    
      📷 写真で投稿（おすすめ）
      
- 商品の写真を送るだけでOK
- 複数枚まとめて送れます
- AIが画像から自動で読み取ります

    
    
      📝 文字で投稿
      
- 商品名・メーカー・品番を書く
- 状態や付属品を書くと精度が上がります

    
  
  写真の方が精度が高いため、できるだけ写真で投稿することを推奨します。


## ステップ2　AIが商品を確認する


投稿するとAIからこのような返答が届きます：
  
```
〜の商品と見たが、合っておるか？
　品名：[商品名]
　メーカー：[メーカー名]
　型番：[型番 または 不明]
```

  
    ※ AIは「北大路魯山人」というキャラクターとして返答します。古風な口調ですが、内容をしっかり確認してください。


## ステップ3　商品情報を確認して返答する


✅ 合っている場合


    `はい` または `OK` と

    スレッドに返信します


    → ステップ4へ進みます
  
  
    ❌ 違う場合


    `違います。〇〇です` と

    補足情報をスレッドに返信します


    → AIが再特定してステップ2に戻ります


⚠️ 「スレッドに返信」とは？

  AIのメッセージの右側に出る「返信」をクリックして、そのメッセージに対して返信することです。

  新しいメッセージとして投稿すると、AIはコマンドと認識できません。


  スマートフォンの場合：メッセージを長押し → 「スレッドで返信」をタップします。


## ステップ4〜5　AIが状態確認を依頼する（通販チャンネル候補のみ）


通販チャンネルが候補の場合、AIからこのような確認依頼が届きます：
  
```
よかろう。では動作・外観の確認をお願いしたい。

　• 電源・動作確認
　• ドア・引き出しなど開閉確認
　• 外観・傷・汚れの確認
　• パーツ・付属品の確認

確認できたら状態ランクを教えてくれ：
　🅢 S：新品・未使用（タグ／箱あり）
　🅐 A：未使用に近い（使用感ほぼなし）
　🅑 B：中古美品（使用感あり・目立つ傷なし）
　🅒 C：中古（使用感・傷・汚れあり）
　🅓 D：ジャンク（動作不良・部品取り）

ランクのアルファベット＋一言コメントで知らせてくれ
例：`B 電源OK・外観に小傷`
```


※ ロット販売・社内利用・スクラップ・廃棄が候補の場合はこのステップをスキップして、直接ステップ7（判定結果）に進みます。


## ステップ6　状態ランクを入力する


アルファベット1文字＋一言コメントをスレッドに返信します。
  
    
      
```
S タグ付き未開封
```

    
    
      
```
B 動作OK 小傷あり
```

    
    
      
```
C 傷汚れあり動作OK
```

    
  
  
    ⚠️ 注意：先頭の1文字が必ずアルファベット（S/A/B/C/D）になるようにしてください。

    ✅ 正しい例：`B 外観に傷あり`

    ❌ 間違い例：`中古美品 外観に傷あり`（→ AIが認識できません）


## ステップ7　AIが判定結果を出力する


このような判定結果が返ってきます：
  
```
━━━━━━━━━━━━━━━━
📦 分荷判定結果
━━━━━━━━━━━━━━━━

📋 アイテム名：パナソニック 洗濯機 NA-F60B14
🏭 メーカー/ブランド：Panasonic
🔢 品番/型式：NA-F60B14
📊 状態ランク：B（中古美品）

─────────────────────────

【第一候補】ヤフオク現行
理由：現行品・動作OK・需要安定

💰 予想販売価格：¥8,000〜¥12,000
🎯 推奨スタート価格：¥7,000
🏆 推奨目標価格：¥10,000
📅 予測在庫期間：〜1ヶ月
⏰ 推奨在庫期限：2026年6月末
📊 市場需要：2（市場拮抗）
🏭 保管コスト概算：¥300
📦 梱包・発送コスト概算：¥2,000
💡 期待ROI：約42%

⭐ 総合スコア：68点
　└ 収益期待：20/30
　└ 在庫回転：18/25
　└ 保管物流効率：16/25
　└ 作業コスト効率：14/20

  ...

━━━━━━━━━━━━━━━━
販売チャンネルを確定するには `第一` `第二` またはチャンネル名を返信してくれ
━━━━━━━━━━━━━━━━
```

  
    ※ 管理番号はこの時点では表示されません。確定コマンドを入力したときに自動で発行されます。


## ステップ8　販売チャンネルを確定する


判定結果のスレッドに、以下のいずれかを返信して確定します：
  
| 入力する内容 | どうなるか |
| --- | --- |
| 第一 または 第1 | AIが提案した第一候補のチャンネルで確定 |
| 第二 または 第2 | AIが提案した第二候補のチャンネルで確定 |
| ヤフオク現行（チャンネル名をそのまま） | そのチャンネルで確定 |
| 確定/ロット販売（確定/＋チャンネル名） | 指定したチャンネルで確定 |

  
    確定すると自動で行われること：
    
- 管理番号の自動発行（通販チャンネルのみ）
- スプレッドシートへの自動記録（全チャンネル共通）
- Monday.comへの在庫登録（通販チャンネルのみ）
- 推奨目標価格¥30,000以上の場合 → 責任者に自動通知

  

  
    📦 まとめ売りチャンネルの場合は追加の選択画面が出ます

    「ヤフオクまとめ」「eBayまとめ」「ロット販売」で確定した場合は、すぐに確定されず3択の選択画面が表示されます。

    → 詳しくは次のセクション「4. まとめ売りチャンネルの確定手順」を参照してください。

# 4　まとめ売りチャンネルの確定手順


⚠️ このセクションは特に重要です。必ず読んでください。

  まとめ売りチャンネル（ヤフオクまとめ・eBayまとめ・ロット販売）では、

  確定時に 追加の選択画面 が表示されます。ここで正しい選択をしないと、やり直しが必要になります。


## 対象となるチャンネル


ヤフオクまとめ
  eBayまとめ
  ロット販売


## なぜ特別な手順があるのか


まとめ売りチャンネルの商品は、1個ずつ管理番号を発行して売るケースと、他の同種商品と合わせてまとめ保管するケースの2通りがあります。


  ほとんどの場合は「まとめ保管」です。管理番号の発行は、浅野から特別な指示があった場合のみ選びます。


## 3択画面の見方と選び方


まとめ売りチャンネルで `第一`（または `第二`）を入力すると、以下の画面が表示されます：


```
━━━━━━━━━━━━━━━━
📦 ヤフオクまとめ（まとめ売り）が選択されました
━━━━━━━━━━━━━━━━

どちらで進めますか？

1️⃣  まとめ保管する（管理番号なし）← ほとんどの場合はこちら

2️⃣  個別に管理番号を発行して確定する

3️⃣  保留にして浅野に相談する

`1` `2` `3` のいずれかを返信してください。
```


| 選択肢 | 意味 | いつ選ぶか |
| --- | --- | --- |
| 1 | まとめ保管（管理番号なし） | ほとんどの場合はこれを選びます。 同カテゴリの商品がまとまったら、後日一括で管理番号を発行して出品します。 |
| 2 | 個別に管理番号を発行して確定 | 浅野から「この商品は個別で管理番号をつけて」と指示があった場合のみ。 自己判断で選ばないでください。 |
| 3 | 保留にして浅野に相談 | 「これまとめでいいのかな？」と迷ったときに選びます。浅野に通知が届きます。 |


⚠️ 迷ったら `1` を選んでください。

  `2`（管理番号発行）を間違えて選ぶと、不要な管理番号が発行されてしまい、キャンセル→やり直しの手間が発生します。


## まとめ保管を選んだ場合の画面


```
━━━━━━━━━━━━━━━━
📦 まとめ保管として記録しました
━━━━━━━━━━━━━━━━

同カテゴリのまとめ対象商品と一緒に保管してください。
まとめ販売が決まった時点で改めて管理番号を発行します。
```


💡 これで完了です。管理番号は発行されず、スプレッドシートにまとめ保管として記録されます。

# 6　管理番号について


## 管理番号とは何か


管理番号は、通販チャンネルに確定した商品に自動で発行される個別の識別番号です。

  この番号を使って、撮影・出品・梱包・出荷の各工程で商品を追跡・管理します。


  非通販チャンネル（ロット販売・社内利用・スクラップ・廃棄）には発行されません。

  まとめ保管（3択で「1」を選んだ場合）にも発行されません。


## 管理番号のフォーマット


2603-0001
  
    
      2603

      年月コード
西暦下2桁 ＋ 月2桁
例）2026年3月 → 2603
    
    
      -
    
    
      0001

      月次連番
その月の何番目か
4桁で自動採番


※ チャンネルを問わず、すべての通販チャンネルで共通の月次連番です。チャンネル別のコード（V/G/M/Eなど）は付きません。


管理番号の読み方　練習例
  
| 管理番号 | 読み方 |
| --- | --- |
| 2603-0001 | 2026年3月・その月の1番目に確定した商品 |
| 2604-0023 | 2026年4月・その月の23番目に確定した商品 |
| 2605-0007 | 2026年5月・その月の7番目に確定した商品 |


## 発行ルールまとめ


管理番号が発行されるケース
    
- eBayシングルで確定
- ヤフオクヴィンテージで確定
- ヤフオク現行で確定
- まとめ系で「2」（個別発行）を選択

    形式：`YYMM-連番4桁`（例：`2603-0001`）
  
  
    管理番号が発行されないケース
    
- ロット販売・社内利用・スクラップ・廃棄
- まとめ系で「1」（まとめ保管）を選択

    データベースには記録されます。

# 7　状態ランクの判断基準


状態ランクは「購入者が受け取ったときにどう感じるか」を基準に判断します。

  迷ったときは、より低いランクを選ぶと安全です（クレーム・返品リスクを防ぐため）。


| ランク | 名称 | 判断基準 | 入力例 |
| --- | --- | --- | --- |
| S | 新品・未使用 | タグ・箱・説明書がすべて揃っており、開封した形跡がない | S タグ付き未開封 |
| A | 未使用に近い | 使った形跡がほとんどない。汚れ・傷が見当たらない | A 使用感なし箱なし |
| B | 中古美品 | 使った形跡はあるが、目立つ傷や汚れはない。動作は正常 | B 動作OK 小傷あり |
| C | 中古 | 使用感・傷・汚れがある。ただし機能的には問題なし | C 傷汚れあり動作OK |
| D | ジャンク | 動作しない・部品が欠品・修理が必要な状態 | D 電源入らず |


## ランク判定 よくある迷いポイント


| 状況 | 判断の目安 |
| --- | --- |
| 小さな傷があるが動作は問題ない | 傷が目立つ → C、目立たない → B |
| 箱はないが未使用に近い状態 | 箱なしでも使用感がなければ A |
| 電源は入るが一部の機能が動かない | 機能に問題あり → D（正直に記録することが重要） |
| 汚れているが洗えばきれいになる | 現状のままでランクを付ける（清掃後ならその状態で） |

  
    💡 ランク判定に迷う場合は、先輩に確認を取るか、写真を撮って相談してください。正直な状態評価が、後のトラブルを防ぎます。


# 8　スコアの見方


## スコアとは何か


スコアは、AIが「この商品をどれだけ効率よく売れるか」を100点満点で評価した数値です。

  スコアが高いほど、利益が出やすく・手間が少なく・早く売れる可能性が高いということです。


| 評価項目 | 配点 | 何を見ているか |
| --- | --- | --- |
| 収益期待 | 30点 | 予想販売価格の高さ、価格の安定性、まとめ売りによる付加価値 |
| 在庫回転 | 25点 | 1週間以内:25点 / 1ヶ月以内:18点 / 3ヶ月以内:10点 / それ以上:3点 |
| 保管・物流コスト効率 | 25点 | 商品のサイズ、発送費、梱包材のコスト、保管期間との組み合わせ |
| 作業コスト効率 | 20点 | 梱包の難しさ、撮影の手間、問い合わせの多さ |


スコアと確信度の目安
  
| スコア | 確信度 | 意味 |
| --- | --- | --- |
| 75点以上 | 高 | AIが自信を持って判定できている。そのまま確定して問題ない可能性が高い |
| 50〜74点 | 中 | 複数のチャンネルが候補になっている。第一・第二候補を比較して判断する |
| 50点未満 | 低 | 判定が難しい商品。先輩や責任者に確認することを推奨 |


## 判定結果に含まれるその他の情報


| 表示項目 | 内容 |
| --- | --- |
| 推奨スタート価格 | ヤフオクに出品するときの開始価格の目安 |
| 推奨目標価格 | 落札を狙う目標価格。¥30,000以上の場合は浅野に自動通知 |
| 推奨在庫期限 | この期限までに出品・処分を目標とする目安日 |
| 保管コスト概算 | 予測在庫期間中の倉庫保管コストの概算 |
| 梱包・発送コスト概算 | 梱包材費＋送料の概算（第一候補チャンネルベース） |
| 期待ROI | 投資対効果の概算（販売価格 ÷ 仕入れ＋経費 − 1） |
| 市場需要（1/2/3） | 1=買い手市場（売りにくい）/ 2=需給拮抗 / 3=売り手市場（売りやすい） |
| 推定内部KW | 社内管理用コード（発送会社・サイズ・価格帯・期待値） |

# 9　自動で記録されること


## 確定すると何が起こるか


確定コマンドを入力すると、以下がすべて自動で行われます。手入力は不要です。


| 記録される内容 | なぜ記録するか |
| --- | --- |
| 日時・担当者 | 誰がいつ作業したかの記録。スタッフ別の実績管理に使われる |
| 管理番号 | その商品を一意に識別するための番号。以降の全工程で使われる |
| 商品名・メーカー・品番 | 同種商品の相場データ蓄積・出品情報の自動生成に使われる |
| 状態ランク | 出品時の価格設定の根拠になる |
| 確定チャンネル / AI候補 | AIの精度検証・販路選択の傾向分析に使われる |
| 予想販売価格・スコア | 在庫の資産評価・売上予測に使われる |
| 推奨スタート価格・目標価格 | 出品担当チャンネルへの価格情報引き継ぎに使われる |
| 推奨在庫期限 | 長期在庫の警告・優先出品判断に使われる |
| 保管コスト・発送コスト概算 | 案件ごとの原価計算・収益分析に使われる |
| 期待ROI | 仕入れ判断の振り返りデータとして蓄積される |
| 推定内部KW | 発送会社・サイズの事前把握に使われる |
| 分荷作業時間（分） | 投稿〜確定コマンドまでの時間を自動計測。1件あたりの作業コスト把握に使われる |


💡 作業時間は自動で計測されています。ゆっくり確認しながら作業して問題ありません。正確な判定の方が大切です。


## 在庫管理の状態遷移（通販チャンネルのみ）


確定時査定待ち → 動作確認後動作確認済み → 撮影後撮影済み → 出品後出品中 → 梱包後梱包済み → 出荷後出荷済み


各工程のスタッフが担当チャンネルで操作するたびに、このステータスが自動で更新されます。

  キャンセルコマンド入力時 → 「キャンセル」に変わります。

# 11　間違えたときの対処方法


間違いは誰にでも起こります。大事なのは、すぐに正しい手順でやり直すことです。

  以下の手順に従えば、自分ひとりでやり直しができます。


## やり直しの全体フロー


削除キャンセルする → 再判定もう一度判定 → 第一 / 第二正しく確定


## ケース1：確定するチャンネルを間違えた


例：ヤフオク現行で確定すべきだったのに、ヤフオクヴィンテージで確定してしまった
  
| 誰が | 操作 |
| --- | --- |
| スタッフ | スレッドに 削除 と入力 |
| AI | ❌ キャンセルしました（管理番号がキャンセルされる） |
| スタッフ | 同じスレッドに 再判定 と入力 |
| AI | 📦 新しい判定結果が表示される |
| スタッフ | 正しいチャンネルで確定（例：確定/ヤフオク現行） |
| AI | ✅ 確定完了（新しい管理番号が発行される） |


## ケース2：まとめ売りで管理番号を発行してしまった


例：ヤフオクまとめで「まとめ保管」にすべきだったのに、「個別発行」を選んでしまった
  
| 誰が | 操作 |
| --- | --- |
| スタッフ | スレッドに 削除 と入力 |
| AI | ❌ キャンセルしました |
| スタッフ | 同じスレッドに 再判定 と入力 |
| AI | 📦 新しい判定結果が表示される |
| スタッフ | 第一（ヤフオクまとめを選択） |
| AI | 📦 3択画面が表示される |
| スタッフ | 1（← 今度は「まとめ保管」を選ぶ） |
| AI | 📦 まとめ保管として記録しました |


## ケース3：まだ確定していない段階で間違えた


例：状態ランクを間違えて入力した / 商品名の確認で「はい」と言ったが実は違った
  
    確定前なら、`再判定` だけでOKです。

    `削除` は不要です。`再判定` と入力すれば、最初からやり直せます。


## やり直し時の注意点


| 注意点 | 理由 |
| --- | --- |
| 必ず 削除 してから 再判定 する | 削除せずに再判定すると、キャンセルされない管理番号がMonday.comに残り続けます |
| 新規投稿ではなく、同じスレッドで操作する | 別のスレッドで操作しても、元のスレッドの確定は取り消されません |
| 再判定の結果が前回と少し変わることがある | 正常です。AIは毎回判定するため多少変わります。候補にチャンネルが出ていればOK |
| 候補に出てこない場合は 確定/チャンネル名 で直接指定 | 例：確定/ヤフオクまとめ と入力すれば直接そのチャンネルで確定できます |


💡 間違いに気づいたらすぐに対処してください。後回しにすると、後工程（撮影・出品）が間違った情報で進んでしまう可能性があります。

  自分で対処が難しい場合は、浅野にSlackで連絡してください。

# 12　困ったときは


## よくある質問


### Q. AIからの返答が来ない・遅い


- しばらく時間がたってから（1〜2分）再度投稿してみてください
- エラーメッセージが出た場合は担当者（浅野）に連絡してください


### Q. 管理番号が発行されなかった


- ロット販売・社内利用・スクラップ・廃棄のチャンネルでは、もともと管理番号は発行されません（正常です）
- まとめ売りチャンネルで「1」（まとめ保管）を選んだ場合も発行されません（正常です）
- ヤフオク系・eBay系のチャンネルで発行されない場合は担当者に連絡してください


### Q. 「スレッドに返信」の方法がわからない


- パソコンの場合：AIのメッセージにマウスを近づけると、右側にメニューアイコンが出ます。「返信」または吹き出しアイコンをクリック
- スマートフォンの場合：メッセージを長押し →「スレッドで返信」をタップ


### Q. 状態ランクの判断に自信がない


- 迷ったときは、より低いランクを選ぶのが安全です
- わからない場合は先輩に商品を見てもらってから入力してください
- 「正直に入力すること」が最も大切です


### Q. 写真が複数ある場合はどうすればよいか


- 複数枚まとめて1回の投稿に添付してください
- AIはすべての画像を見て判定します


### Q. 判定結果に納得できない


- 再判定 コマンドを使って追加情報を入力すると、精度が上がることがあります
- それでも迷う場合は @浅野儀頼 相談（または「相談があります」「相談です」）と送って相談モードを使ってください
- AIの提案はあくまで「参考意見」です。最終判断はスタッフが行います


### Q. まとめ売りの3択で何番を選べばいいかわからない


- 迷ったら 1（まとめ保管）を選んでください。これが一番安全です
- 2（個別発行）は浅野から指示があった場合のみ選びます
- 判断に自信がなければ 3（保留・浅野に相談）を選んでください


### Q. 間違えて確定してしまった・やり直したい


- スレッドに 削除 と入力 → 再判定 → 正しく確定し直せます
- 詳しい手順はセクション11「間違えたときの対処方法」を参照してください
- 自分で対処が難しければ、浅野にSlack DMで連絡してください


### Q. 浅野さんに個別に相談したい


- スレッドに @浅野儀頼 相談があります と送ると、浅野さんに通知が届きます
- その後そのスレッドはボットが反応しなくなるので、自由に相談してください


## 問い合わせ先


| 困りごとの種類 | 連絡先 |
| --- | --- |
| システムの不具合・エラー | 浅野儀頼に連絡 |
| 操作方法がわからない | 先輩スタッフ または 浅野に確認 |
| 判定の妥当性に疑問がある | 再判定 コマンドを使うか、@浅野儀頼 相談 で直接相談 |
| 確定を間違えた・やり直したい | 削除 → 再判定 で自力対処。難しければ浅野に連絡 |
| ¥30,000以上の高額案件 | 確定時に浅野へ自動通知が届きます（手動連絡不要） |


わからないことは、何でも聞いてください。

  迷ったまま進めるより、確認してから進める方が、チーム全体にとって良い結果につながります。

  一緒に良い職場をつくりましょう。
